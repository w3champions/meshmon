## Recurring patterns observed

1. **Stale rust-analyzer diagnostics**: every implementer report has been followed by editor diagnostics that look like real errors but are stale snapshots. **Always trust `cargo build` / `cargo xtask test` over the diagnostic notification**. When in doubt, dispatch a 1-shot verifier subagent.

2. **The `Endpoint::Agent` vs `Endpoint::CandidateIp` pool model**: bit Phase E and Phase F. The fix is `Endpoint::CandidateIp { ip: a.ip }` for intermediaries (LegLookup keys are `(Agent, Ip)`). Phase E uses dual-form (both Agent and CandidateIp per agent); Phase F's main loop uses dual-form too. Architectural limitation: pure `Agent‚ÜíAgent` legs unresolvable, so 2-hop X-routes for diversity/optimization don't work (only pure-mesh 2-hop tiebreaker works).

3. **Plan inaccuracies**:
   - `measurements.source` column claimed but doesn't exist (Phase C).
   - Pre-existing PATCH dismissal claimed but didn't exist (Phase B).
   - `hostname` column should never be on `campaign_evaluation_candidates` (Phase G).
   - `meshmon/docs/campaigns/` path is actually `docs/campaigns/`.
   - `qualifies_first` and `coverage_weighted_ping_*` sorts on edge_pairs deferred (need JOIN to candidates).
   - `destination_agent_id` filter on edge_pairs deferred.

4. **Implementer overconfidence**: Phase G implementer claimed 75 tests pass ‚Äî they actually had 1 test failure that aborted the run, leaving 406 tests unrun. Always have spec reviewer run `cargo xtask test` independently.

## Known limitation to document

The 2-hop X-routes (`A‚ÜíX‚ÜíY‚ÜíB` and `A‚ÜíY‚ÜíX‚ÜíB`) for diversity/optimization are NOT enumerable by the current `LegLookup` model. The pure-mesh 2-hop tiebreaker (`qualifies_under_optimization_v2`) IS correctly implemented. The `winning_x_position = Some(2)` field is reachable for EdgeCandidate but not for diversity/optimization. This should be noted in the spec/architecture doc during Phase U, and `winning_x_position = Some(2)` for diversity/optimization can be considered a future enhancement requiring LegLookup model extension.
